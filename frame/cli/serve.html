# 部署

我们推荐前后端分离的开发实践，利用工具提供的 mock 能力可以轻松地模拟数据接口。

dist 目录中包含的构建内容是纯静态的页面资源，可以被部署到任何一个静态资源服务器中，但要确保正确的 base 路径。

# 本地部署验证

dist 目录需要启动一个 HTTP 服务器来访问 (除非你已经将 base 配置为了一个相对的值)，所以以 file:// 协议直接打开 dist/index.html 是不会工作的。

工具集成了一个 Node.js 静态资源服务器，执行 serve 命令可以快速启动，并对指定的目录提供静态资源服务。具体如下：

$ hui serve -h
hui serve <dir> [options]

使用内置服务器为指定目录提供静态资源服务

选项：
  --help, -h  显示帮助信息  [布尔]
  --base      指定静态资源服务的虚拟路径  [字符串] [默认值: "/"]
  --log       开启访问日志  [布尔] [默认值: false]
  --port, -p  指定服务器端口  [字符串] [默认值: "9000"]

WARNING

执行 serve 命令必须指定目录 dir，如果指定的目录不存在时将会报错

# 选项

# --base

为静态资源服务指定一个基础路径。具体如下：

$ hui serve ./dist --base /foo/

WARNING

特别地，如果在 hui.config.js 中指定了 base 参数，在执行 serve 命令部署静态资源服务时，必须指定该选项。

# --log

开启访问日志，具体如下：

$ hui serve ./dist --log

# --port, -p

指定服务器端口，默认是 9000，具体如下：

$ hui serve ./dist -p 8080

# history 路由模式

在 hui.config.js 中配置 mode 为 history 时，dist 目录是无法搭配简单的静态资源服务器进行部署的。例如，如果在 views 目录下存在一张 todos/business.vue 视图页面，开发服务器已经配置了相应的 localhost:8000/todos/business 响应，但是一个为生产环境构建架设的简单的静态服务器会却会返回 404。

为了解决这个问题，你需要配置生产环境服务器，将任何没有匹配到静态文件的请求回退到 index.html。以 nginx 为例：

location / {
  try_files $uri $uri/ /index.html;
}

TIP

以上示例假设你在根目录服务这个应用。如果想部署到一个子目录，你需要配置 hui.config.js 中的 base 路径。你还需要把上述示例中的根目录调整成为子目录 (即用 /name-of-your-subfolder/index.html 替换掉 /index.html)。

TIP

可能你已经注意到了，工具集成的静态资源服务已经做了这样的处理。

# CORS

如果前端静态内容是部署在与后端 API 不同的域名上，你需要适当地配置 CORS。

TIP

基于 HUI CLI 提供的静态资源服务已经做了 CORS 的处理。
