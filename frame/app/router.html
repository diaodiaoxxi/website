# 路由

路由是统一框架 2.x 很重要的一部分，hui-core 在 vue-router 的基础上进行了封装。目前支持两种路由模式，约定式路由和配置式路由。

约定式路由可以覆盖大多数的应用场景，一般情况下，这应该是业务系统首选的路由方案。在约定式路由模式下，整个应用的路由系统由工具接管，约定 src/views 目录为路由视图目录，在开发阶段，工具会自动监控该目录并生成路由脚本。整个过程无需开发者主动介入，开发者所有要做的事情只是，创建一个路由视图组件，然后进行业务编码。

# 约定式路由

开启约定式路由的方式也非常简单，只要在 hui.config.js 中设置 autoRouting 为 true 即可。在应用创建的过程中，工具也会以命令行交互的方式提醒开发者是否开启 autoRouting 模式。

与约定式路由相配合，在 hui.config.js 中也可以设置路由前缀 base 和历史记录模式 mode。

# 基础路由

假设 views 目录结构如下：

views
|── user
|   |── index.vue
|   └── one.vue
└── index.vue

那么，框架自动生成的路由配置如下：

// src/.hui/router.js
initRouter(
  [
    {
      path: "/",
      component: RouterLayout,
      children: [
        {
          name: "index",
          path: "",
          component: Index,
        },
        {
          name: "user-index",
          path: "user",
          component: UserIndex,
        },
        {
          name: "user-one",
          path: "user/one",
          component: UserOne,
        },
      ],
    },
  ],
  { base: "/", mode: "hash" }
);

# 动态路由

以下划线 _ 作为前缀的 Vue 文件或目录会被解析为动态路由。

假定目录结构如下：

views
|── _slug
|   |── comments.vue
|   └── index.vue
|── users
|   └── _id.vue
└── index.vue

生成的对应的路由配置如下：

// src/.hui/router.js
initRouter(
  [
    {
      path: "/",
      component: RouterLayout,
      children: [
        {
          name: "index",
          path: "",
          component: Index,
        },
        {
          name: "users-id",
          path: "users/:id?",
          component: UsersId,
        },
        {
          name: "slug-index",
          path: ":slug",
          component: SlugIndex,
        },
        {
          name: "slug-comments",
          path: ":slug/comments",
          component: SlugComments,
        },
      ],
    },
  ],
  { base: "/", mode: "hash" }
);

# 嵌套路由

在框架 2.x 中，创建内嵌子路由，需要添加一个 Vue 文件，同时添加一个与该文件同名的目录用来存放子视图组件。

WARNING

必须在父组件(.vue 文件) 内增加 <router-view /> 用于显示子视图内容。

假设文件结构如下：

views
|── users
|   |── _id.vue
|   └── index.vue
└── users.vue

自动生成的路由配置如下：

// src/.hui/router.js
initRouter(
  [
    {
      path: "/",
      component: RouterLayout,
      children: [
        {
          name: "index",
          path: "",
          component: Index,
        },
        {
          name: "users-index",
          path: "users",
          component: UsersIndex,
        },
        {
          name: "users-id",
          path: "users/:id?",
          component: UsersId,
        },
      ],
    },
  ],
  { base: "/", mode: "hash" }
);

# 动态嵌套路由

在动态路由下配置动态子路由，可以通过添加以下划线 _ 开头的目录实现。

views
|── _category
|   |── _subCategory
|   |   |── _id.vue
|   |   └── index.vue
|   |── _subCategory.vue
|   └── index.vue
|── _category.vue
└── index.vue

自动生成的路由配置如下：

// src/.hui/router.js
initRouter(
  [
    {
      path: "/",
      component: RouterLayout,
      children: [
        {
          name: "index",
          path: "",
          component: Index,
        },
        {
          path: ":category?",
          component: Category,
          children: [
            {
              name: "category-index",
              path: "",
              component: CategoryIndex,
            },
            {
              path: ":subCategory?",
              component: CategorySubCategory,
              children: [
                {
                  name: "category-subCategory-index",
                  path: "",
                  component: CategorySubCategoryIndex,
                },
                {
                  name: "category-subCategory-id",
                  path: ":id?",
                  component: CategorySubCategoryId,
                },
              ],
            },
          ],
        },
      ],
    },
  ],
  { base: "/", mode: "hash" }
);

# 非路由组件

特殊地，以双下划线 __ 开头，并且以双下划线 __ 结尾的 .vue 文件（比如 __foo__.vue）不会被路由引擎处理，这可以用来添加一些公共的组件逻辑。

WARNING

这并不是一个推荐的方式，建议在 components 目录下完成这部分工作。

# 特殊的路由约定

特殊地，__404__.vue 将作为 404 路由，它对应的路由配置如下：

{
  "name": "__404__",
  "path": "*",
  "component": __404__
}

# 布局组件

约定 src/layouts 目录下的 .vue 文件在应用内将作为共享的布局组件。在视图组件中，可以选择指定 layout 选项，默认值为 default。

比如，存在 layouts/foo.vue 内容如下：

<!-- layouts/foo.vue -->
<template>
  <div>
    <h1>Foo Layout</h1>
    <router-view />
  </div>
</template>

存在 views/index.vue 内容如下：

<!-- pages/index.vue -->
<template>
  <p>index.vue</p>
</template>

<script>
export default {
  // You can specify layout component name here (default value is 'default')
  layout: "foo",
};
</script>

最终，index.vue 将为渲染为：

<div>
  <h1>Foo Layout</h1>
  <p>index.vue</p>
</div>

# 配置式路由

关闭 autoRouting 就会自动进入配置式路由模式。在该模式下，开发者需要主动处理路由关系，包括添加路由和设置路由参数。具体如下：

import hCore from "hui-core";
import routes from "@/router";

const app = hCore();

// 调用 addRoutes 方法可以添加路由
app.addRoutes(routes);

app.start(() => {
  app.log("app started successfully");
});

// src/router/index.js
export default [
  {
    name: "index",
    path: "/",
    component: Index,
    children: [
      {
        name: "index-home",
        path: "home",
        component: IndexHome,
      },
    ],
  },
];

function Index() {
  return import(/* webpackChunkName: "js/views/index" */ "@/views/index.vue");
}

function IndexHome() {
  return import(/* webpackChunkName: "js/views/index-home" */ "@/views/index/home.vue");
}
