# SelectTree2.0

# SelectTree2.0 概述

::: tip 概要说明

  • selectTree2.0需要搭配select2.0使用
  • 可选择的数据结构是一个树形结构时，可以使用 TreeSelect，例如公司层级、学科系统、分类目录等等

:::

# 基础用法

::: tip 用法说明

defaultExpandAll

  • 初始化默认展开所有树节点大数据场景不推荐使用，会造成性能卡顿

expandLevel

  • 初始化默认展开几级树节点，优先级：defaultExpandAll > expandLevel

:::

# 父子组件不联动

::: tip 用法说明 checkStrictly

  • 多选模式下，checkStrictly=true父子节点选中状态不再关联

:::

# 自定义过滤

::: tip 用法说明 filterNodeMethod

  • 按需筛选树节点（高亮），返回 true

:::

# itemHeight

::: tip 用法说明

  • 当下拉列表的高度改变时，需要传入tree每个item的高度（包括加上margin）
  • 每个下拉节点的高度必须一致，treenode每个item的高度若参差不齐或者itemHeight不一致，则滚动计算失误
  • 若用户在外部自定义修改了每个节点的高度，且不设置itemHeight属性，会出现selectTree定位失效等问题

:::

# v-model回显策略

::: tip 用法说明 showCheckedStrategy

  • v-model变量显示的值的策略方式
  • SHOW_ALL：显示所有选中节点(包括父节点)(默认)
  • SHOW_PARENT：只显示父节点(当父节点下所有子节点都选中时)
  • SHOW_CHILD：只显示子节点

:::

# 鼠标提示

  • renderTextTips个性化定义提示文本；使用方法参考tree组件使用方法

showCheckedStrategy v-model变量显示的值的策略方式
SHOW_ALL：显示所有选中节点(包括父节点)(默认)
SHOW_PARENT：只显示父节点(当父节点下所有子节点都选中时)

# 事件 Events

方法名 说明 回调参数
on-expand 树节点展开/收缩时出发 1. 返回已经展开的值 expandedKeys
2. 返回 eventobj
详情可参考tree
on-inside-select 内部改变值时触发 1. 返回已经选中的值
2. 返回 eventobj

::: tip on-inside-select

单选

  • 选中下拉树触发

多选

  • 选中下拉树节点触发
  • 输入框头部点击tag头×清除图标触发
  • 输入框通过backspace删除图标触发

注意：点击输入框清除图标×时不触发，当前操作在select(父组件)有单独的触发事件on-clear-select事件

:::

# 键盘

::: tip 键盘

键盘按下触发keydown时；（具体使用逻辑，参考tree键盘事件）

  • 当点击节点，默认focus初始化不聚焦节点为-1
  • 支持上下键聚焦节点，逻辑参考tree
  • focusIndex聚焦的节点，支持左右键展开节点，左键收起，右键展开，逻辑参考tree
  • 当focusIndex不是-1，有聚焦节点，enter键可以选中节点
  • 多选时，ctrl+a/mac支持command+a 支持全选，ctrl+d/mac支持command+d 支持全不选（具体使用逻辑，参考tree键盘事件）

:::
