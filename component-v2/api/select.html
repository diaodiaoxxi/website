# Select2.0

# Select 概述

概要说明

  • 当前select控件完全不同于1.0select控件，2.0select控件注重于针对选择器头部交互的整合
  • 2.0select不能单独使用，必须搭配适配与下拉框内部控件的搭配使用，目前只开放2.0selectTree组件搭配使用（其他控件待开发），详细请看示例
  • 搭配selectTree2.0使用，生成selectTree下拉树控件

# 清除图标

用法说明 clearable

  • clearable:true 是否始终显示图标；默认鼠标hover显示;
  • 当v-model的值是空时，图标不显示（只有在有值时，显示图标）
  • 当设置 disabled || readonly 时，不显示清除图标

# 禁用、只读

用法说明 禁用disabled、只读readonly

公共功能

  • 可以通过外部动态设置值改变
  • 鼠标hover，展示禁用图标
  • 边框不高亮
  • 清除图标不展示(clearable)
  • 点击不展开下拉框（禁止点击）
  • 多选时，有tag时，鼠标hover，可以展示更多
  • 多选时，tag图标中的×清除按钮不显示
  • 多选时,键盘Backspace清除tag选项，失效
  • 调用onFocus方法，聚焦失效
  • 调用onBlur方法，失焦失效
  • on-click事件，点击头部，不触发

区别

  • disabled 禁止输入框值的复制
  • readonly 允许输入框值的复制

# 多选

用法说明

multiple

  • 设置multiple（多选），开启多选，输入框以tag内容显示

filterable(可搜索场景下)

  • 点击搜索，更新下拉列表，点击选择下拉项，输入框值不清空，值处于选中状态
  • 点击外部选项，组件失焦，输入框值清空，下拉列表显示成初始值的数据

maxTagCount

  • 设置多少个tag开始收起通过hover显示
  • 设置maxTagCount，设置最多展示多少个后，收起，以poptip形式展示，默认为1（默认tag需要在一行显示）
  • 设置maxTagCount，当选中值>200时，不展示更多tag（目前设计如此，后续可能会改变）

maxTagPlaceholder

  • 设置maxTagPlaceholder，自定义设置隐藏 tag 时显示的内容，当绑定的是string时，eg: :maxTagPlaceholder="还有 ${value2.length - 3} 个"，输入框显示 还有XXX个

# 多选头类型

用法说明 multHeadType

  • 设置multHeadType,自定义设置输入框中，多选内容的展现形式（默认：tag）
  • 设置multHeadType="text",输入框内容以逗号隔开的字符串内容显示
  • 设置multHeadType="text"，当输入框内容文本过多时，输入框内容以…形式展示
  • 设置multHeadType="text"，filterable可搜索状态下，当输入框input聚焦时，文本内容呈现浅色透明状态，当输入内容query值非空时，文本消失

# 粘贴赋值

用法说明

  • selectPaste开启粘贴值，可以成功回显（选中下拉项）
  • 注意： selectPaste支持回显，会触发两次on-change
  • 输入框粘贴触发on-paste事件；参数 pasteText：当前赋值的内容；{event:当前粘贴事件}）
  • 用户可通过绑定on-paste事件，对v-model绑定的变量进行外部重新赋值，当前场景不会触发两次on-change事件

注意点：

  • 前提：filterable可搜索场景
  • 注意拷贝内容的多余空格可能会导致回显失败

# 下拉框宽度

用法说明（dropWidth、widthAdaption、maxDropWidth）

widthAdaption=true，开启下拉框宽度自适应场景下

  • 设置dropWidth值为下拉框宽度的min-width，最小宽度值
  • dropWidth为自适应场景下的最小宽度，若 dropWidth > maxDropWidth， 下拉框宽度 为 maxDropWidth，自适应将失效
  • maxDropWidth受输入框自身宽度限制，若设置maxDropWidth < 输入框自身宽度时，maxDropWidth取输入框自身宽度时 （即：maxDropWidth >= 输入框自身宽度）

widthAdaption=false默认，关闭下拉框宽度自适应场景下

  • 未设置dropWidth（常规场景下），下拉框宽度为输入框宽度
  • 设置dropWidth，则下拉框宽度为dropWidth宽度
  • 设置maxDropWidth最大宽度失效

# 方法

用法说明 onFocus、onBlur

onFocus

  • 通过 this.refs.xxx.onFocus()/this.refs.xxx.onFocus(true)调用，,实现输入框的聚焦 ，输入框呈现聚焦状态，默认展开下拉框
  • 通过 this.refs.xxx.onFocus(false)调用，实现输入框的聚焦 ，输入框呈现聚焦状态,不展开下拉框
  • 通过 this.refs.xxx.onFocus()调用，触发on-focus事件
  • 通过 this.refs.xxx.onFocus()调用，触发表单on-form-focus的校验（默认）
  • 通过 this.refs.xxx.onFocus(false)调用，filterable可搜索时，大部分输入按键（目前除'Tab'外，后期可能会改），可展开下拉框
  • 通过 this.refs.xxx.onFocus()调用，当外部设置 disabled 或者 readonly, 调用失效

onBlur

  • 通过 this.refs.xxx.onBlur()/this.refs.xxx.onBlur(true)调用,手动取消输入框焦点，输入框是去焦点状态，可搜索场景下，输入框回显（单选）
  • 通过 this.refs.xxx.onBlur(false)调用,手动取消输入框焦点，输入框失去焦点状态,下拉框不收起，可搜索状态下，输入框不回显（单选）
  • 通过 this.refs.xxx.onBlur()调用，触发on-blur事件
  • 通过 this.refs.xxx.onBlur()调用，触发表单校验 on-form-blur （默认）
  • 通过 this.refs.xxx.onBlur()调用，当外部设置 disabled 或者 readonly, 调用失效

# FAQ

# select2.0单独使用，为什么不生效

  • 首先，通用select2.0，达到了交互的统一，由于1.0的历史原因，导致表单控件类组件，不同的组件，同一场景下，可能存在同一事件，触发机制不一样的问题

  • select2.0跟1.0select完全不同，大家不要把这两个组件混为一谈。2.0的select组件更加注重于针对select选择器类控件交互头的整合，因此其不能单独使用，必须搭配适配与下拉框内部控件的搭配使用，目前只开放2.0selectTree组件搭配使用（其他控件待开发eg: option2.0、selectTable2.0等）

  • 使用select2.0，搭配使用不同的下拉框内部控件，做到了交互的完全统一。（on-change、表单校验等触发机制的统一）
