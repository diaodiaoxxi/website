# Tree2.0 树形控件

# Tree2.0 概述

::: tip 概要说明

  • 文件夹、组织架构、生物分类、国家地区等等，世间万物的大多数结构都是树形结构。使用树控件可以完整展现其中的层级关系，并具有展开收起选择等交互功能。
  • 支持大数据场景（设置 virtual）
  • 支持通过绑定变量，实现数据的双向绑定
  • tree2.0 不同于 tree1.0 的使用，请耐心看完示例

:::

# 展开/收缩

::: tip 用法说明

  • :expandedKeys.sync 实现双向绑定

  • expandLevel：初始化默认暂开几级树节点

  • defaultExpandAll：初始化默认展开所有树节点； 大数据场景不推荐使用 优先级：defaultExpandAll > expandLevel

  • 方法 nodeExpand()

  • on-expand 事件，外部通过:expandedKeys.sync绑定的变量更改，不触发

  • 优先级：初始化 defaultExpandAll > （expandLevel、expandedKeys）；当设置 expandLevel、expandedKeys，展开两个属性都符合的节点

:::

# 过滤、筛选节点

::: tip 用法说明

  • filterValue 根据 filterValue 绑定的变量值筛选树节点（高亮显示符合条件的节点）

  • isTreeFilter 搜索时，直接过滤树节点

  • filterNodeMethod 自定义按需筛选树节点（高亮），返回 true；搭配 filterValue 使用

  • filterType 设置当isTreeFilter=true时，设置过滤方式。只要针对过滤场景下，复选框联动以及去全选、全不选的逻辑计算

# 禁用 disabled

::: tip 用法说明 (tree 设置 disabled、node 节点设置 disabled)

  • 禁止点击复选框选中、禁止点击文本

  • 支持节点展开/收缩；

  • 支持异步加载节点

  • 支持外部通过变量设置值来选中复选框、文本（checkedKeys、selectedKeys）

  • 禁止 on-click、on-right-click、on-double-clickdeng 等事件触发

  • 外部调用方法 nodeSelect()、nodeCheck()，不生效；nodeExpand()有效；

  • 节点 node 设置 disabled, 则当前节点不参与复选框联动计算

  • 键盘 enter 选中节点，不生效，左右键展开、收缩节点生效

  • 鼠标 hover 不显示...更多图标

# selectable

::: tip 用法说明 (tree 设置 selectable: false、 节点设置 selectable: false)

  • 节点文本禁止选中

  • 外部调用 nodeSelect()，不生效

  • 不支持文本节点选中，支持复选框选中

  • 点击文本转化为复选框选中的逻辑处理（eg：选中复选框，相应触发 on-check 等事件 `）

tree 设置selectable: false场景下

  • 表示所有节点文本禁止点击选中；外部设置 selectedKeys, 文本也禁止生效；

  • node节点对象设置selectable:true，不代表当前节点可以文本选中，代表当前节点不支持点击文本选中复选框的操作；

  • node节点设置selectable:false，代表当前节点当前不可点击文本选中，点击文本选中事件，转变为checkbox选中事件的逻辑处理

:::

# titleMaxWidth、isBoxRight

::: tip 用法说明

  • titleMaxWidth: 设定每个节点的 title 文字最大宽度，当文本超出显示省略号，鼠标悬浮显示全部文本

blockNode=true场景下

  • 当 titleMaxWidth < 文本区域块最大宽度，文本超出其本身块的宽度时，宽度取到文本区域块的宽度，出现省略号

  • 当 titleMaxWidth > 文本区域块的宽度，文本超出其本身块的宽度时，宽度取 titleMaxWidth 的宽度，溢出出现省略号

isBoxRight: true

  • 当 titleMaxWidth < 文本区域块最大宽度，isBoxRight 复选框的位置取文本区域块的最右边位置；文本溢出出现省略号

  • 当 titleMaxWidth > 文本区域块最大宽度，isBoxRight 复选框的位置取 titleMaxWidth 的位置；文本溢出出现省略号

:::

# 大数据（virtual）

::: tip 大数据场景

  • 大数据场景下，设置 virtual 属性

  • itemHeight 每行节点 li 的高度（当外部修改样式，改变其高度时，必须设置该属性；每个节点的高度必须一致）

  • maxHeight tree 的最大可视区域高度（默认：500）; (eg: 当 tree 需要改变高度时，必须设置改属性)

  • 不推荐使用 defaultExpandAll，节点展开过多，会造成卡顿

  • 不推荐 expandLevel 层级设置过高

  • 根节点数据过多时，建议只展示根节点或展开一层

  • 注意：tree 大数据性能，跟展开节点数目有关；请谨慎

  • 在大数据场景下，单一数场景下，滚动树节点，不涉及 checked、selected、expanded、loading 等计算时,可以设置 onlyScroll 来提高滚动的效率 :::

# transferScrollOptions

::: tip 用法说明

滚动场景下，针对dropdown处理方式

  • 动态设置当 tree 滚动时，内部的 dropdown 下拉框是否关闭的问题（详见api说明）

  • IE下，keep模式支持不友好，默认会改为hide模式

:::

# FAQ

# 数据量较大时，页面卡顿

  • tree2.0(uTree)支持大数据，需要开启虚拟滚动（virtual=true）

# 虚拟滚动的限制

  • 针对数据量比较大的情况，推荐设置virtual属性，开启虚拟滚动

  • 虚拟滚动通过在仅渲染可视区域的元素来提升渲染性能。但是同时由于不会渲染所有节点，所以无法自动拓转横向宽度（比如超长 title 的横向滚动条）

  • 在大数据场景下，绑定expandedKeys属性时，expandedKeys展开的keys,会设计到计算性能等问题，在push时i，需要谨慎对待。进来避免push不必要的子节点

# disabled节点在树中的关系是什么

  • Tree 通过传导方式进行数据变更。无论是展开还是勾选，它都会从变更的节点开始向上、向下传导变化，直到遍历的当前节点是 disabled 时停止。因而如果控制的节点本身为 disabled 时，那么它只会修改本身而不会影响其他节点。举例来说，一个父节点包含 3 个子节点，其中一个为 disabled 状态。那么勾选父节点，只会影响其余两个子节点变成勾选状态。勾选两个子节点后，无论 disabled 节点什么状态，父节点都会变成勾选状态

  • 这种传导终止的方式是为了防止通过勾选子节点使得 disabled 父节点变成勾选状态，而用户无法直接勾选 disabled 父节点更改其状态导致的交互矛盾。如果你有着自己的传导需求，可以通过 checkStrictly自定义勾选逻辑

# 针对isTreeFilter节点过滤场景下，联动计算逻辑

  • 针对isTreeFilter节点过滤场景下，过滤存在两种计算方式，根据filterType属性来区别

  • filterType=filter,过滤类型为真实的过滤tree，你可以理解为内部以filterValue关键字过滤生成了新的tree，内部的checkbox联动关系处理，全选、全不选都是以新生成的tree来计算的

  • filterType=search，过滤类型为查询节点，过滤只是当做查询节点来看(而不是实实在在的过滤tree，内部默认生成一颗新的tree)，内部的checkbox联动关系处理，全选、全不选都是整个tree来计算的

# 为什么修改的treeData数据内部节点的属性值，视图没有更新

  • tree内部没有对treeData做深监听操作，若直接针对内部节点做修改，是不会触发视图更新的

  • 可以通过调用this.$refs.xxx.propsForceUpdate('treeData')来实现视图更新

# 为什么针对treenode节点做操作，内部属性节点值不会同步到treeData原始数据

  • tree2.0设计思路，做到了treeData跟内部操作的数据，完全解耦的状态，因此其调用开放的方法，回参包含一些当前节点的item项时，直接通过item修改属性的值，是不会同步到treeData原始数据
