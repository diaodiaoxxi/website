# 系统安全

现代浏览器的设计目标是安全、快速和稳定

  • 浏览器内核是由网络进程、浏览器主进程和 GPU 进程组成的

  • 渲染内核就是渲染进程

# 安全沙箱

将渲染进程和操作系统隔离的这道墙就是我们要聊的安全沙箱。其实就是阻止 渲染进程 可以直接访问或者修改 操作系统的 数据，通过渲染进程 访问或者修改 操作系统时，需要通过 浏览器内核来实现，并且将结果通过 IPC 的形式 返回给 渲染进程

安全沙箱就是渲染进程和操作系统之间建一道墙，渲染进程由于存在漏洞被黑客攻击，但由于这道墙，黑客就获取不到渲染进程之外的任何操作权限

# 安全沙箱如何影响各个模块功能

浏览器内核、渲染 内核是如何通讯的

  1. 持久存储

    • 存储 Cookie 数据的读写
    • 读写一些缓存文件，比如网络文件缓存的读取。

  2. 网络访问

    • 渲染进程内部也是不能直接访问网络
    • 浏览器内核在处理 URL 请求之前，会检查渲染进程是否有权限请求该 URL，比如检查 XMLHttpRequest 或者 Fetch 是否是跨站点请求，或者检测 HTTPS 的站点中是否包含了 HTTP 的请求

  3. 用户交互

    • 应用程序可以在 窗口句柄 上进行绘制和接收键盘鼠标消息
    • 渲染进程不能直接访问窗口句柄

# 站点隔离（Site Isolation）

所谓站点隔离是指 Chrome 将同一站点（包含了相同根域名和相同协议的地址）中相互关联的页面放到同一个渲染进程中执行

# 你认为安全沙箱能防止 XSS 或者 CSRF 一类的攻击的吗？为什么？

不能， 安全沙箱主要是渲染进程和操作系统之间的隔离； XSS、CSRF主要是攻击网页、服务器的漏洞，跟浏览器本身没有关系，这两个攻击其实都是在渲染进程中的
