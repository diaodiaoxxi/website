# PWA

# 什么是pwa 渐进式网页应用

一套理念，可以渐进式增强web优势，并通过技术的手段来渐进式缩短与本地应用和小程序的距离。 基于这套理念下的技术都可以归类为 PWA

  • web开发者角度

    • pwa就是一个渐进式的过度方法，让web应用逐步具有本地应用的能力（逐渐本地应用化的能力）
    • 降低站点改造的代价，使站点逐步支持各项新技术，而不是一步到位

  • 技术的角度

    • pwa就是一个渐进式演化的过程，在技术层面一点点演进
    • 比如逐渐提供更好的设备特性支持，不断的让页面加载速度越来越快，不断实现本地应用化的能力

# web网页缺点

  • 缺少离线缓存功能
  • 缺少消息推送功能
  • 缺少一级页面访问入口

通过 Service Worker 来实现离线缓存和消息推送的功能，通过引入 Manifest.json来解决一级入口的问题

# Service Worker

  • 主要思想：在页面和网络之前添加一个拦截器（代理服务器），用来实现拦截和缓存的请求

  • Service workers 本质上充当 Web 应用程序、浏览器与网络（可用时）之间的代理服务器 创建有效的离线体验，它会拦截（http）网络请求并根据网络是否可用采取来适当的动作、更新来自服务器的的资源

  • 针对消息推送，HTTP 采用的是明文传输信息，存在被窃听、被篡改和被劫持的风险， Service Worker 采用 HTTPS 协议，因为采用 HTTPS 的通信数据都是经过加密的，即便拦截了数据，也无法破解数据内容，而且 HTTPS 还有校验机制，通信双方很容易知道数据是否被篡改

# manifest.json

PWA 还提供了 manifest.json 配置文件，可以让开发者自定义桌面的图标、显示名称、启动方式等信息，还可以设置启动画面、页面主题颜色等信息

# PWA的优势

  • 可以将app的快捷方式放置到桌面上，全屏运行，与原生app无异
  • 能够在各种网络环境下使用，包括网络差和断网条件下，不会显示undefind
  • 推送消息的能力
  • 其本质是一个网页，没有原生app的各种启动条件，快速响应用户指令

# PWA存在的问题

  • 支持率不高:现在ios手机端不支持pwa，IE也暂时不支持
  • Chrome在中国桌面版占有率还是不错的，安卓移动端上的占有率却很低
  • 各大厂商还未明确支持pwa
  • 依赖的GCM服务在国内无法使用
  • 微信小程序的竞争

# 原生App

  • 开发成本高(ios和安卓)
  • 软件上线需要审核
  • 版本更新需要将新版本上传到不同的应用商店
  • 想使用一个app就必须去下载才能使用，即使是偶尔需要使用一下下
