# 早期单进程

不稳定 不流畅 不安全 脚本或者插件会让单进程浏览器变卡顿外，页面的内存泄漏也是单进程变慢的一个重要原因。

# 多进程浏览器时代

最新的 Chrome 浏览器包括：1 个浏览器（Browser）主进程、1 个 GPU 进程、1 个网络（NetWork）进程、多个渲染进程和多个插件进程。

带来的问题：

更高的资源占用 更复杂的体系架构

chrome使用多个进程来隔离不同的标签页

  • 进程之间是不共享资源和地址空间的；每个进程之间不太会出现互相影响的安全问题

  • 多个线程，是共享着相同的资源和地址空间的，所以线程之间就有可能会存在恶意修改或者获取非授权数据的复杂性安全问题

  • Browser进程

浏览器的主进程（负责调控、主控），只存在一个 负责浏览器界面显示，与用户交互，前进、后退 负责各个页面间的管理，创建和销毁其他进程 网络资源管理，下载等

  • 网络进程

主要负责页面的网络资源加载，之前是作为一个模块运行在浏览器进程里面的，直至最近才独立出来，成为一个单独的进程。

  • 第三方插件

每种类型的插件是一个进程，当使用时创建

  • GPU进程

3D制作 硬件加速

  • Renderer进程：浏览器渲染进程（浏览器内核）
