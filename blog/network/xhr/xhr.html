# 说明

# XHR、AJAX、Axios、Fetch

# XHR

xhr（XMLHttpRequest）对象，用于与服务器端的交互。

优点

  • 在不刷新页面的情况下，请求特定的URL，获取数据，达到局部刷新页面的效果

缺点

  • 使用步骤繁杂，不简洁明了。
  • 早期的IE浏览器有自己的内置对象，需要针对ie做兼容性处理

# AJAX

优点

  • 本质上是对xhr对象的封装，简化了使用
  • 增大了对jsonp的支持，解决了部分跨域问题

缺点

  • 请求过多，并且有依赖关系，容易造成地狱回调

# Axios

基于promise的http库，可用于浏览器和node.js。其本质上也是对xhr对象的封装，只是其是基于对promise的实现、

本质是 基于 promise 对 xhr对象的封装

优点

  • 支持请求，响应拦截
  • 支持请求取消
  • 支持链式调用
  • 支持Promise api
  • 转换请求数据和响应数据

缺点

  • 只支持现代化浏览器

# Fetch

fetch api是JavaScript的api(内置对象)，用于访问＆操作http管道，比如请求、响应等。同时也提供了全局的fetch方法，可以有效简洁的解决跨域资源请求的问题

优点

  • 基于promise的实现，支持async/await
  • 支持链式调用
  • 替代了XHR

缺点

  • 支持网络请求错误，比如500，400之类的不会报reject错误
  • 不支持abort中断、不支持超时
  • 不支持获取原生请求进度，xhr的可以通过onprogress获取当前进度情况
  • 默认不带cookie
