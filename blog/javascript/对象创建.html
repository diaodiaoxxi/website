# 对象创建

# 工厂模式

缺点：无法识别对象所属类型，所有的实例都指向同一个原型


(function(){

  function createPerson(name) {
    let obj = new Object()
    obj.name = name
    obj.getName = function() {
      return this.name
    }
    return obj
  }
  var preson1 = createPerson('Lily')
  var preson2 = createPerson('Bob')

})()

# 构造函数模式

  • 优点：可以识别特定的对象类型
  • 缺点：每次创建实例，每个方法都要被重新创建一次（比较占内存）

 (function(){

  function Person(name) {
    this.name = name
    this.getName = function() {
      return this.name
    }
  }
  var preson1 = new Person('Lily')
  var preson2 = new Person('Bob')

})()

# 构造函数模式优化

  • 优点：可以识别特定的对象类型；解决了每个方法都要被重新创建的问题
  • 缺点：如果在构造函数中，需要创建很多方法，那必须在全局环境中定义相对应数量的方法，
  • 只是在但是这些方法定义的Person构造函数创建的对象上才能使用，针对代码使用上有迷惑性

 (function(){

  function Person(name) {
    this.name = name
    this.getName = getName
  }
  function getName() {
    return this.name
  }
  var preson1 = new Person('Lily')
  var preson2 = new Person('Bob')
})()

# 原型模式

  • 优点：方法不会出现重新创建的问题
  • 缺点：
    • 所有的方法、属性都共享，针对有些想要私有化的属性，无法做到私有化
    • 初始化无法传参

 (function(){
  function Person() {
  }
  Person.prototype.name = 'Lily'
  Person.prototype.getName = function() {
    return this.name
  }
  var preson1 = new Person()
})()

# 原型模式优化

  • 优点：直接使用对象字面量赋值，针对原型需要重新定义赋值，方便写法
  • 缺点:
  • constructor添加与否：未添加丢失了构造函数
  • 原先的缺点任然保留

 (function(){
  function Person() {
  }
  Person.prototype = {
    // constructor: Person,
    name: 'Lily',
    getName: function() {
      return this.name
    }
  }
  var preson1 = new Person()

})()

# 组合模式（构造函数、原型模式）

  • 优点：对象属性个性化定制其私有化、共享性（该共享的共享，该私有的私有，使用最广泛的方式）

 (function(){
  function Person(name) {
    this.name = name
  }
  Person.prototype.getName = function() {
    return this.name
  }
  var preson1 = new Person()
})()

# 动态原型模式

  • 注意：使用动态原型模式时，不能用对象字面量重写原型
  • new 的实现步骤：
    1. 首先新建一个对象
    2. 然后将对象的原型指向 Person.prototype
    3. 然后 Person.apply(obj)
    4. 返回这个对象
  • 当实例化构造函数时，Person.prototype已经指向了obj对象的原型，
  • 当后续执行 Person.prototype = {...}对象字面量的 形式赋值，直接覆盖了覆盖 Person.prototype，并不会更改obj实例的原型的值，obj依旧指向之前原型的值

 (function(){
  function Person(name) {
    this.name = name
    if (typeof this.getName !== 'function') {
      Person.prototype = {
        constructor: Person,
        getName: function () {
            console.log(this.name);
        }
      }
      return new Person(name);
    }
  }
  var preson1 = new Person()
})()

# 寄生构造函数模式（寄生在构造函数的一种方法）

  • 挂羊头买狗肉（工厂模式）
  • 缺点：
    1. 挂羊头买狗肉（工厂模式），创建的实例使用 instanceof 都无法指向构造函数
    1. 也无法识别对象所属类型

 (function(){
  function Person(name) {
    var o = new Object();
    o.name = name;
    o.getName = function () {
        console.log(this.name);
    };
    return o;
  }
  var preson1 = new Person('Lily')
  console.log(preson1 instanceof Person)  //  false
  console.log(preson1 instanceof Object) //  true
})()

# 稳妥构造函数模式

  • 所谓稳妥对象，指的是没有公共属性，而且其方法也不引用 this 的对象
  • 与寄生构造函数模式有两点不同：
      1. 实例不使用this
      1. 不使用new操作符创建对象
  • 缺点：稳妥构造函数模式也跟工厂模式一样，无法识别对象所属类型

(function(){
 function person(name) {
   let o = new Object()
   o.getName = function() {
     console.log(name)
   }
   return o
 }
 var preson1 = person('Lily')

})()
